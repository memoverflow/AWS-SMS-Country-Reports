## 一、AWS SMS 服务概述与通用设置流程

### 1.1 服务名称

AWS 的 SMS 服务目前称为 **AWS End User Messaging SMS**（前身为 Amazon Pinpoint SMS / Amazon SNS SMS）。支持 SMS、MMS、RCS 和语音消息，覆盖全球 32 个 AWS 区域。

### 1.2 沙箱模式（Sandbox）

新账户默认处于沙箱模式，限制如下：

| 限制项 | 默认值 |
|--------|--------|
| 月度 SMS 消费限额 | $1.00 USD |
| 月度 MMS 消费限额 | $1.00 USD |
| 可发送的目标号码 | 最多 10 个已验证号码 |

### 1.3 升级到生产环境（Production）

**步骤：**
1. 前往 AWS Support Center 创建工单：
   `https://console.aws.amazon.com/support/home#/case/create?issueType=service-limit-increase`
2. 填写信息：
   - **Category**: Account and Billing
   - **Service**: AWS End User Messaging SMS (Pinpoint)
   - **Quota**: SMS Production Access
   - **New quota value**: 1
3. 工单描述中必须包含：
   - 网站/应用链接
   - 消息类型（OTP / Promotional / Transactional）
   - 目标国家/地区
   - 用户 opt-in 流程说明
   - 消息模板示例
4. **如果在多个 AWS Region 使用，需为每个 Region 分别提交**
5. 通常 24 小时内初次响应

### 1.4 号码类型说明

| 号码类型 | 说明 | 月费（参考） | 申请周期 |
|---------|------|-------------|---------|
| **10DLC** | 美国10位长码，A2P主要方式 | $1/号码 + $10/Campaign | 4-8周 |
| **Short Code（短码）** | 4-7位数字，高吞吐量 | 美国$995/月，英国$1,500/月 | 8-12周 |
| **Toll-Free（免费号码）** | 888/877等开头，仅限美国/加拿大 | $2/月 | 较快 |
| **Long Code（长码）** | 标准电话号码格式 | 因国家而异（$2-$27/月） | 1-3周 |
| **Sender ID** | 字母数字品牌名称（如"MyBrand"） | 通常免费 | 因国家而异 |

### 1.5 消息计费规则

- **按消息片段（segment）计费**，无论是否成功送达
- GSM 7-bit 编码：单条最多 160 字符，拼接时每段 153 字符
- Unicode 编码（含中文/emoji）：单条最多 70 字符，拼接时每段 67 字符
- **拼接消息按每段单独计费**
- 总费用 = AWS 消息费 + 运营商费（carrier fee）

#### 1. 美国 (US) +1

| 项目 | 详情 |

| 长码/10DLC | 支持 |
| Toll-Free | 支持 |
| Sender ID | **不支持** |

**10DLC 注册流程（必须）：**

| 步骤 | 内容 | 美国公司 | 国际公司 |
|------|------|---------|---------|
| 1. 品牌注册 | 公司法律全称、EIN、地址等 | 1-2个工作日 | 最多3周 |
| 2. 品牌审核（推荐） | 第三方验证，费用$40 | 1-2个工作日 | 最多3周 |

| 4. 号码关联 | 购买号码并绑定Campaign | 约14天 | 约14天 |

**吞吐量限制（未审核）：**
- AT&T：75条/分钟
- T-Mobile：2,000条/天（同公司所有账户共享）

**核心法规：TCPA（电话消费者保护法）**
- 未经同意发送，每条可罚 $500-$1,500
- 必须获得明确 opt-in 同意
- 禁止内容：S.H.A.F.T.（色情、仇恨、酒精、枪支、烟草）、赌博、高风险金融等
- 禁止使用免费短链接（tinyurl、bitly 等）

**Sender ID 注册（UK MEF SMS Sender ID Protection Registry）：**

分两种情况：

**情况A：MEF 受保护的 Sender ID**
1. 下载 AWS LOA 模板（`AWS_Protected_Sender_ID_Letter-of-Authorisation_UK.zip`）

4. 在 AWS 控制台提交注册

**情况B：非 MEF 受保护的 Sender ID**
- 直接在 AWS 控制台填写注册表单，无需 LOA

**所需材料：** 公司名称、税号、网站、地址、联系人信息、LOA（如适用）、消息样本1-3条

**不注册后果：** 消息可能被拦截

## 五、最佳实践与合规建议

### 5.1 Opt-in（用户同意）管理

- **所有国家均要求获得用户明确同意**
- 确认消息示例：`"回复 YES 加入 XX公司 通知服务。每月约2条。可能产生短信费用。回复 HELP 获取帮助，回复 STOP 退订。"`
- Opt-in 必须披露：品牌名称、消息频率、产品描述、退订方式、费用说明
- 不同组织间不共享 opt-in 列表

### 5.2 Opt-out（退订）处理

- 支持 STOP/HELP 关键词的国家：自动处理
- 不支持双向短信的国家（阿根廷、哥伦比亚等）：必须提供替代退出方式（网页链接、邮箱等）
- 退订请求须在合理时间内处理

### 5.3 消息类型分离

| 类型 | Transactional（事务性） | Promotional（营销型） |
|------|----------------------|---------------------|
| 用途 | OTP、订单确认、告警 | 优惠、推广、品牌宣传 |
| 要求 | 不得混入营销内容 | 必须获得明确书面同意 |
| 建议 | 使用独立号码/短码 | 使用独立号码/短码 |

**重要：** 事务性消息中混入营销内容在多个国家（土耳其、沙特、阿联酋等）会被严格处罚。

### 5.4 发送时间

- 仅在收件人当地工作时间发送（通常 9:00-20:00）
- 避免周日和国定假日（部分国家法律禁止）
- 注意目标国家时区差异

### 5.5 URL 使用

- **禁止使用免费短链接服务**（tinyurl、bitly 等），运营商会过滤
- 使用品牌自有域名
- 土耳其（2026年4月起）对 URL 有额外限制
- 新西兰含 URL 的消息需白名单审批

### 5.6 冗余设计

- 在多个 AWS Region 配置 SMS
- 使用多种号码类型增加冗余
- 号码不可跨 Region 复制，需在每个 Region 分别申请
